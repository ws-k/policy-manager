개선점 요약

총 12개 개선 항목 — Critical 2 / High 4 / Medium 4 / Low 2

Critical 정책 목록 페이지 헤더 부재

헤더 좌측이 <div/> 빈 요소. 페이지 타이틀 / 요약 통계가 없어 사용자가 현재 위치와 데이터 현황을 즉시 파악할 수 없음.

Critical 정책 상세 액션 버튼 과부하

published 상태에서 6개 버튼이 동시 노출(내보내기, 버전비교, 공개중, 링크복사, 새버전, 게시취소). 시각적 위계 없이 수평 배치되어 주요 행동 식별이 어려움.

High 기능맵 카드 시각적 잡음

각 카드에 편집/연결/삭제 버튼 3개가 항상 노출. 특히 삭제 버튼이 빨간 배경으로 항상 보여 오조작 위험. 핵심 정보(기능명+정책)보다 액션이 시각적으로 더 강함.

High accent 색상 단조로움

--color-accent: #111827(검정) 하나로 모든 주요 요소를 처리. 도메인 칩, 사이드바 active, 배지가 동일 색상이라 기능적 구분이 어려움.

High 퍼블릭 뷰 디자인 시스템 미적용

public-view에서 하드코딩 bg-gray-50, text-gray-900 등 CSS 변수 미사용. 관리자와 공개 뷰 사이에 디자인 시스템 단절.

High 정책-기능 연결 가시성 부족

정책 목록에서 각 정책에 몇 개의 기능이 연결되어 있는지 확인 불가. 정책 관리의 핵심 가치인 "기능 커버리지"가 목록 수준에서 보이지 않음.

Medium 정책 상세 네비게이션 빈약

"← 목록으로" 텍스트 링크만 존재. 빵크럼 없어 정책 > 도메인 > 현재 문서의 위계적 위치 파악 불가. 상위 도메인으로의 직접 이동 경로 없음.

Medium TOC 현재 위치 표시 없음

PolicyTOC가 스크롤 추적(Intersection Observer) 없이 정적 목록. 현재 읽고 있는 섹션이 어디인지 시각적 피드백이 없어 긴 문서에서 위치 감각 상실.

Medium 기능맵 미연결 상태 시각적 미분화

정책 연결된 카드와 미연결 카드가 동일한 실선 테두리. "정책 미연결" 텍스트만으로는 빈 상태가 충분히 드러나지 않아 커버리지 갭을 놓칠 수 있음.

Medium 도메인 칩에 개수 미표시

정책 목록 필터 도메인 칩에 해당 도메인의 정책 수가 없어 필터링 전 데이터 분포를 예측할 수 없음. 불필요한 클릭 발생.

Low 사이드바 아이콘 유니코드 문자 사용

네비게이션 아이콘이 유니코드 문자(⊞, ≡, ◈, ◎)로 구현. 시각적 일관성과 크기 제어가 어렵고, OS/브라우저별 렌더링이 다를 수 있음.

Low StatusBadge 컴포넌트 중복 정의

StatusBadge가 policy-list-client.tsx와 policy-detail-client.tsx에 각각 별도로 정의. padding 값이 미세하게 다름(px-2 py-0.5 vs px-2.5 py-1). 공통 컴포넌트로 추출 필요.
